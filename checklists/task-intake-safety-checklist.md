# Чек-лист приёма OSINT-задачи

Перед началом работы агент должен быстро проверить задачу.

## 1. Цель

- [ ] Цель сформулирована ясно.
- [ ] Понятно, какой результат нужен: справка, отчёт, таблица, вывод, план, анализ изображения.
- [ ] Есть границы: что искать и что не искать.

## 2. Объект исследования

Разрешённые объекты:

- [ ] организация;
- [ ] сайт;
- [ ] публичный проект;
- [ ] публичный документ;
- [ ] новостное событие;
- [ ] изображение без задачи идентификации частного лица;
- [ ] учебный синтетический пример.

Рискованные объекты:

- [ ] частное лицо;
- [ ] автомобильный номер с целью найти владельца;
- [ ] адрес;
- [ ] телефон;
- [ ] аккаунт, который нужно деанонимизировать;
- [ ] маршрут или регулярное местоположение.

Если отмечен рискованный объект, нужно перейти к безопасной альтернативе или отказать.

## 3. Источники

- [ ] Используются только открытые и законно доступные источники.
- [ ] Не используются утечки.
- [ ] Не используются закрытые базы.
- [ ] Не обходятся авторизация, капча, платные ограничения или приватность.
- [ ] Не выполняется активное агрессивное сканирование.

## 4. Персональные данные

- [ ] Не собираются лишние персональные данные.
- [ ] Не публикуются адреса, телефоны, документы, маршруты.
- [ ] При необходимости данные обезличиваются.
- [ ] В отчёте есть только минимально необходимая информация.

## 5. Итог

- [ ] Указан уровень уверенности.
- [ ] Есть ссылки или описания источников.
- [ ] Отделены факты от предположений.
- [ ] Указаны ограничения.
- [ ] Нет вредных инструкций.

## Быстрое решение

```text
РАЗРЕШЕНО: продолжить работу.
ОГРАНИЧИТЬ: выполнить только безопасную часть.
ОТКАЗАТЬ: запрос направлен на вред, доксинг, слежку или обход приватности.
```
