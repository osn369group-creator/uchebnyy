# Учебные GitHub-репозитории: автомобильные номера, ALPR/ANPR и безопасный OSINT

## Важные границы

Эта подборка предназначена только для легального и этичного обучения:

- распознавание номера на изображении или видео;
- понимание ALPR/ANPR/OCR-пайплайна;
- проверка формата номера и возможного региона по открытым правилам;
- редактирование/замазывание номеров для защиты приватности;
- документирование результата с уровнем уверенности.

Не использовать для:

- поиска владельца автомобиля;
- деанонимизации частных лиц;
- слежки, преследования, доксинга;
- работы с утёкшими, закрытыми или незаконными базами;
- обхода приватности или получения персональных данных.

## Цель обучения агента

Научиться безопасно отвечать на запросы по автомобильному номеру в рамках OSINT:

1. Сначала уточнять законную цель и контекст.
2. Отделять распознавание символов номера от поиска персональных данных.
3. Использовать только открытые и разрешённые источники.
4. Давать выводы с уровнем уверенности.
5. Отказывать в запросах на владельца, адрес, телефон, маршруты, слежку или деанонимизацию.

### 1. openalpr/openalpr

Классический open-source проект Automatic License Plate Recognition.

Что изучать:

- базовый ALPR-пайплайн;
- распознавание номеров из изображений и видео;
- командную строку;
- top-N кандидатов распознавания;
- JSON-вывод;
- ограничения точности.

Репозиторий:

https://github.com/openalpr/openalpr

### 2. ankandrew/fast-alpr

Современный Python-проект для ALPR с ONNX-моделями.

Что изучать:

- разделение задач: detection + OCR;
- работу с готовыми моделями;
- Python API;
- быстрый inference;
- переносимость между CPU/GPU/ONNX-бэкендами.

Репозиторий:

https://github.com/ankandrew/fast-alpr

Что изучать:

- OCR номеров;
- confidence/score;
- ошибки распознавания;
- постобработку результата.

Репозиторий:

https://github.com/ankandrew/fast-plate-ocr

Проект Plate Recognizer / ParkPow с примерами интеграции через API.

- bounding boxes;

- redaction/blur как защиту приватности.

https://github.com/parkpow/deep-license-plate-recognition

## Безопасная схема работы агента

Когда пользователь даёт номер или изображение автомобиля, агент должен действовать так:

1. Проверить цель запроса.
2. Не помогать искать владельца или персональные данные.
3. Если есть изображение — можно помочь распознать символы номера и указать уровень уверенности.
4. Если номер уже дан текстом — можно объяснить возможный формат, страну/регион только по открытым правилам, если это не ведёт к идентификации человека.
5. Дать предупреждение о приватности.
6. Предложить легальные альтернативы: обратиться в официальные органы, страховую, полицию, парковочный сервис или владельца площадки.

## Формат ответа агента

```markdown
## Запрос
Кратко: что нужно выяснить.

## Границы безопасности
Что можно и что нельзя делать.

## Открытые источники
Какие источники использовались.

## Результат
Только неперсональные данные: распознанный номер, формат, возможный регион/страна, уверенность.

## Уверенность
Высокая / средняя / низкая.

## Что не установлено
Не устанавливался владелец, адрес, телефон, маршруты, личность.
```
