# Safe OSINT Toolkit — разрешённые инструменты агента

Этот файл описывает инструменты и рабочие приёмы, которые можно использовать в публичном учебном репозитории.

## 1. Поиск и сбор открытых данных

Разрешено:

- Поиск по официальным сайтам.
- Поиск по публичным реестрам, если они законно доступны и не раскрывают лишние персональные данные.
- Поиск по новостям, публикациям, документации, пресс-релизам.
- Поиск по GitHub-репозиториям, issues, pull requests, README и лицензиям.
- Анализ публичных страниц организаций, продуктов и проектов.

Запрещено:

- Закрытые базы, утечки, купленные базы, боты пробива.
- Попытки получить данные, которые не предназначены для публичного доступа.
- Поиск персональных данных частного лица без ясного законного основания.

## 2. Проверка источников

Инструменты:

- Карточка источника.
- Сравнение минимум двух независимых источников.
- Проверка даты публикации и актуальности.
- Проверка автора, владельца домена, организации и контекста.
- Отметка уровня уверенности: высокий, средний, низкий.

## 3. Анализ изображений и OCR

Разрешено:

- Извлечение видимого текста.
- Распознавание номерного знака как текста на изображении.
- Оценка качества изображения и уверенности распознавания.
- Замазывание или маскирование номеров, лиц, адресов и других идентификаторов.
- Описание видимых объектов без установления личности владельца.

Запрещено:

- Поиск владельца автомобиля по номеру.
- Связывание номера с адресом, телефоном, маршрутом или частным лицом.
- Слежка по изображениям или построение маршрутов.

## 4. Анализ GitHub-репозиториев

Разрешено:

- Анализ README, структуры проекта, зависимостей, лицензии.
- Поиск признаков активности проекта.
- Проверка issues и pull requests.
- Оценка рисков использования библиотеки.
- Поиск документации и примеров.

Запрещено:

- Использовать код для обхода ограничений, взлома, кражи данных или скрытого наблюдения.
- Сохранять вредные инструкции в учебный репозиторий.

## 5. Отчётность

Каждый результат должен содержать:

- Цель анализа.
- Границы и ограничения.
- Источники.
- Факты.
- Гипотезы.
- Уровень уверенности.
- Что не проверено.
- Рекомендованные безопасные следующие шаги.

## 6. Безопасные альтернативы рискованным запросам

Вместо пробива:

- Подготовить официальное обращение.
- Зафиксировать факт в отчёте.
- Обезличить материалы.
- Проверить только публичный формат данных.
- Объяснить, какие органы или организации имеют право запрашивать закрытую информацию.
