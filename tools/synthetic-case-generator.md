# Synthetic Case Generator — генератор учебных OSINT-кейсов

Этот инструмент описывает, как создавать безопасные учебные кейсы без реальных персональных данных.

## Цель

Создавать вымышленные сценарии для тренировки анализа, проверки источников, OCR, документации и отчётности.

## Правила генерации

1. Не использовать реальные ФИО, номера, телефоны, адреса, email, аккаунты, фотографии или VIN.
2. Все идентификаторы должны быть явно учебными.
3. Номера транспорта должны быть помечены как `SAMPLE`, `TEST`, `DEMO` или `FICTIONAL`.
4. Даты, города, компании и домены должны быть вымышленными или зарезервированными для примеров.
5. Кейсы не должны учить преследованию, доксингу или обходу приватности.

## Типы кейсов

### 1. Проверка утверждения

Пример:

> Вымышленная компания `North River Robotics` заявляет, что получила сертификацию ISO-9001. Нужно проверить, какие открытые источники могли бы подтвердить или опровергнуть это утверждение в учебной среде.

Навыки:

- Формулирование гипотез.
- Поиск источников.
- Разделение фактов и предположений.
- Отчёт.

### 2. Анализ GitHub-проекта

Пример:

> Есть учебный репозиторий `fictional-org/vision-demo`. Нужно оценить README, лицензию, активность, зависимости и риски.

Навыки:

- Анализ структуры репозитория.
- Проверка лицензии.
- Оценка активности.
- Выводы для пользователя.

### 3. OCR учебного изображения

Пример:

> На синтетическом изображении виден текст `DEMO-PLATE-001`. Нужно извлечь текст, оценить уверенность и подготовить обезличенный отчёт.

Навыки:

- OCR.
- Уровень уверенности.
- Обезличивание.
- Границы анализа.

### 4. Публичная организация

Пример:

> Нужно составить карту публичных источников о вымышленной организации: официальный сайт, документация, публикации, GitHub, новости.

Навыки:

- Карта источников.
- Приоритизация.
- Триангуляция.

```md
## Статус

## Данные на входе

## Разрешённые методы

## Запрещённые методы

## Задачи агента

## Ожидаемый отчёт

## Критерии качества
```

## Контроль качества

Перед сохранением кейса проверь:

- Нет ли реальных персональных данных.
- Нет ли инструкций по пробиву или слежке.
- Понятна ли учебная цель.
- Есть ли безопасный ожидаемый результат.
