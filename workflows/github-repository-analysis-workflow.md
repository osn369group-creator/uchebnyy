# Workflow: анализ GitHub-репозитория

Цель: расширить возможности агента по изучению публичных и доступных GitHub-репозиториев.

## Что можно анализировать

- README;
- документацию;
- структуру каталогов;
- исходный код;
- зависимости;
- issues;
- pull requests;
- releases;
- GitHub Actions workflows;
- лицензии;
- признаки активности проекта.

## Запрещено

- использовать найденные секреты, токены или ключи;
- помогать эксплуатировать уязвимости;
- публиковать приватные данные;
- искать личные данные разработчиков вне контекста их публичной профессиональной активности;
- использовать анализ для атак.

Если агент случайно обнаружил секрет или чувствительные данные, он не должен их повторять. Нужно указать, что найден возможный секрет, и рекомендовать ответственное раскрытие владельцу репозитория.

## Шаг 1. Быстрая карточка проекта

```markdown
# Карточка репозитория

- Репозиторий:
- Назначение:
- Основной язык:
- Лицензия:
- Последняя активность:
- Уровень зрелости:
- Основные компоненты:
- Быстрый вывод:
```

## Шаг 2. Структура

Определить:

- точку входа;
- основные модули;
- тесты;
- документацию;
- конфигурации;
- CI/CD;
- примеры использования.

## Шаг 3. Зависимости

Проверить файлы вроде:

- `package.json`;
- `requirements.txt`;
- `pyproject.toml`;
- `go.mod`;
- `Cargo.toml`;
- `pom.xml`;
- `Dockerfile`;
- `.github/workflows/*`.

Фиксировать только безопасные наблюдения: типы зависимостей, назначение, потенциальные зоны внимания. Не давать инструкций по эксплуатации уязвимостей.

## Шаг 4. Issues и PR

Оценить:

- какие проблемы обсуждаются;
- активность поддержки;
- повторяющиеся баги;
- открытые вопросы;
- качество коммуникации;
- наличие дорожной карты.

## Шаг 5. Вывод для обучения

```markdown
## Что агент выучил из репозитория

- Техническая тема:
- Полезные подходы:
- Ограничения проекта:
- Что можно применить в учебной базе:
- Что нельзя применять:
- Следующие упражнения:
```

## Уровни анализа

### Базовый

Назначение, установка, запуск, основные файлы.

### Средний

Архитектура, зависимости, тесты, workflow.

### Продвинутый

Качество поддержки, риски сопровождения, сравнение с аналогами, безопасные улучшения.
