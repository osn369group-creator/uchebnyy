# Workflow: безопасный анализ публичного веб-следа

Цель: расширить возможности агента по анализу публичных сайтов, организаций, проектов и брендов без сбора частных персональных данных.

## Разрешённые объекты анализа

- публичные организации;
- компании;
- сайты;
- домены;
- открытые проекты;
- публичные продукты;
- публичные документы;
- официальные страницы.

## Запрещённые объекты и цели

- частные лица без явного согласия;
- поиск домашнего адреса;
- поиск личного телефона;
- поиск маршрутов и привычек;
- деанонимизация;
- сбор данных для преследования;
- использование утечек или закрытых баз.

## Шаг 1. Описание объекта

Записать:

- название объекта;
- официальный сайт, если известен;
- страна/регион, если это публично указано;
- цель исследования;
- ограничения.

## Шаг 2. Карта открытых страниц

Проверить только публичные страницы:

- главная страница;
- about / о компании;
- contacts / контакты;
- legal / privacy / terms;
- blog / news;
- docs / documentation;
- careers / вакансии;
- публичные пресс-релизы.

## Шаг 3. Публичные технические признаки

Разрешено анализировать на высоком уровне:

- доменное имя;
- публичные DNS-записи без активного сканирования;
- видимые технологии сайта, если они раскрыты страницей;
- публичные репозитории;
- публичную документацию API;
- robots.txt и sitemap.xml как обычные публичные файлы.

Не выполнять агрессивное сканирование, подбор директорий, эксплуатацию уязвимостей, обход ограничений или нагрузочные проверки.

## Шаг 4. Репутационные источники

Искать открытые и законные источники:

- официальные публикации;
- СМИ;
- документация;
- публичные реестры организаций, если они законно доступны;
- отзывы как слабый сигнал, а не как доказательство;
- GitHub, npm, PyPI и другие публичные экосистемы при анализе проектов.

## Шаг 5. Матрица выводов

```markdown
| Наблюдение | Источник | Тип источника | Уверенность | Комментарий |
|---|---|---|---|---|
| ... | ... | официальный/вторичный/слабый | высокая/средняя/низкая | ... |
```

## Шаг 6. Итог

Итог должен содержать:

- что подтверждено;
- что не подтверждено;
- какие источники использованы;
- что требует дополнительной проверки;
- уровень уверенности;
- ограничения исследования.

## Формулировки осторожности

Использовать:

- «публичные источники указывают»;
- «по состоянию на дату проверки»;
- «не удалось подтвердить»;
- «это предположение, а не факт»;
- «требуется независимое подтверждение».

Не использовать без доказательств:

- «точно мошенники»;
- «владелец — ...»;
- «скрываются»;
- «доказано», если нет первичных подтверждений.
