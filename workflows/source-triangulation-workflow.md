# Workflow: проверка источников и триангуляция

Цель: научить агента проверять информацию по нескольким независимым открытым источникам и не делать выводы по одному неподтверждённому сигналу.

## Когда использовать

- при проверке новости;
- при анализе утверждения;
- при сравнении данных из разных публикаций;
- при подготовке отчёта;
- при работе с публичной организацией, проектом, продуктом или событием.

## Шаг 1. Сформулировать проверяемое утверждение

Плохой вариант:

> Компания подозрительная.

Хороший вариант:

> Нужно проверить, существует ли организация, какие публичные источники подтверждают её деятельность и есть ли независимые сведения о проектах.

## Шаг 2. Разделить данные на категории

- факт;
- источник;
- дата публикации;
- автор или организация;
- первичный или вторичный источник;
- уровень доверия;
- что именно подтверждается;
- что остаётся неподтверждённым.

## Шаг 3. Найти независимые подтверждения

Минимальный стандарт:

- один первичный источник, если он доступен;
- один независимый вторичный источник;
- отсутствие серьёзных противоречий.

Более высокий стандарт:

- несколько независимых источников;
- понятная дата;
- ясный контекст;
- подтверждение ключевых деталей, а не только общего тезиса.

## Шаг 4. Оценить источник

Вопросы:

- Кто опубликовал информацию?
- Есть ли у источника интерес исказить данные?
- Можно ли проверить дату?
- Ссылается ли источник на документы?
- Повторяет ли он чужой материал без проверки?
- Есть ли опровержения или исправления?

## Шаг 5. Уровень уверенности

### Высокий

Данные подтверждены первичными или несколькими независимыми надёжными источниками.

### Средний

Есть подтверждения, но часть деталей остаётся непроверенной.

### Низкий

Есть только один источник, источник слабый, данные устарели или есть противоречия.

## Шаг 6. Итоговый формат

```markdown
## Проверяемое утверждение

...

## Найденные источники

| Источник | Тип | Что подтверждает | Дата | Надёжность |
|---|---|---|---|---|
| ... | первичный/вторичный | ... | ... | высокая/средняя/низкая |

## Вывод

...

## Уровень уверенности

Высокий / средний / низкий

## Ограничения

...
```

## Безопасность

Не использовать утечки, закрытые базы, данные с признаками незаконного происхождения, персональные данные частных лиц и материалы, полученные через обход доступа.
